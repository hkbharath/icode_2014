UVCEians have a nature of speaking in cryptic languages.They tend to even change 
an ordinary set of numbers into a cryptic set. With N = 5, an random arrangement 
of 5 numbers might look like 2,3,4,5,1.

UVCEians with their urge of being cryptic, represent this arrangement of numbers
as 5,1,2,3,4  where the ith number is the position of 
the integer i in the original arrangement. However this technique 
was found to have a glitch!!

Some arrangements never changed, even after the crypting technique was applied. 
These were called "AMBIGUOUS" arrangement. 
for eg:the arrangement 1,4,3,2 is ambiguous, because its cryptic arrangement 
is also the same.

Your job is to find out whether a given set of numbers are ambiguous or not??

INPUT
The input contains several test cases.
The first line of each test case contains an integer N.
Then a arrangement of the integers 1 to N follows in the 
next line. There is exactly one space character between consecutive integers.

You can assume that every integer between 1 and N appears exactly 
once in the permutation.

The last test case is followed by a zero.

OUTPUT
For each test case output "ambiguous", if the arrangement is ambiguous,
else print "not ambiguous" in a single line.

SAMPLE INPUT
4
1 4 3 2
5
2 3 4 5 1
1
1
0

SAMPLE OUTPUT

ambiguous
not ambiguous
ambiguous
